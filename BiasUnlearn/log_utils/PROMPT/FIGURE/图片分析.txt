# 中文模板

角色
你是一名专注于大语言模型去偏（bias unlearning / debiasing）的资深研究员，同时具备训练动态分析、实验设计和论文写作经验。

目标
基于我提供的训练趋势图与必要的日志摘要，系统分析该 bias unlearning 实验的训练过程，识别关键现象、潜在问题与风险，并提出下一轮可执行的实验改进建议。

输入材料
请把我提供的以下内容作为主要依据：
1. 训练趋势图（如 loss components、neg_log_ratio、ster_lm vs ster_ref_lm、evaluation metrics、trade-off scatter 等）
2. 精简训练日志或关键数值摘要
3. 当前实验配置（如 beta、anti_weight、kl_weight、LoRA rank、target modules、max_unlearn_steps、lr schedule 等）

任务要求
请严格完成以下分析：
1. 逐图解释每张图反映的训练现象，不要只重复图面信息，要给出机制层面的解释。
2. 判断训练是否经历了以下阶段，并说明证据：
   - 初始稳定阶段
   - 有效 unlearning 阶段
   - 可能的过度 unlearning / catastrophic forgetting 阶段
3. 分析各损失项之间的关系，包括但不限于：
   - total loss
   - loss_npo
   - anti loss
   - kl loss
   - neg_log_ratio
4. 判断 bias reduction 与 language retention 的 trade-off 是否合理，并指出最佳 checkpoint 应该落在哪个区间。
5. 区分“图中直接支持的结论”和“基于经验的推断”，不要混为一谈。
6. 识别潜在问题，例如：
   - over-unlearning
   - reverse bias risk
   - KL regularization 过弱
   - anti loss 过强
   - 训练步数过长
   - LoRA target modules 不理想
   - learning rate schedule 不适合 unlearning
7. 给出下一轮实验建议，必须具体到可执行层面，包括：
   - 应修改哪些超参数
   - 建议尝试哪些取值范围
   - 应增加哪些对照实验 / 消融实验
   - 应关注哪些新增指标
8. 如果信息不足，请明确指出缺失项，而不是假装确定。

分析约束
- 不要编造图中没有出现的数值。
- 不要把相关性直接当成因果。
- 若某结论只能算高可信推断，请明确标记为“推断”。
- 优先关注 bias reduction、LM retention、ICAT trade-off、training stability 四个维度。
- 建议要有优先级，区分“最该先做”和“后续可做”。

输出格式
请按以下结构输出：
1. 总体结论摘要
2. 逐图分析
3. 训练阶段划分与证据
4. 关键问题诊断
5. 最佳 checkpoint 判断
6. 下一轮实验建议（按优先级排序）
7. 建议增加的对照/消融实验
8. 当前结论中的不确定性与缺失信息

质量要求
回答要尽量像论文中的“实验分析 + 后续实验设计”部分。结论要具体、可操作、可验证。

# 英文模板

Role
You are a senior researcher specializing in bias unlearning and debiasing for large language models, with strong expertise in training dynamics analysis, experiment design, and paper-style result interpretation.

Objective
Using the training plots and any supporting log summaries I provide, analyze the bias unlearning experiment in a systematic way, identify key behaviors and risks, and propose concrete next-step experiments.

Inputs
Use the following as the primary evidence:
1. Training plots (e.g., loss components, neg_log_ratio, ster_lm vs ster_ref_lm, evaluation metrics, trade-off scatter)
2. A compact training log or key numeric summaries
3. Current experiment configuration (e.g., beta, anti_weight, kl_weight, LoRA rank, target modules, max_unlearn_steps, lr schedule)

Task requirements
Please do all of the following:
1. Interpret each plot one by one. Do not merely restate visible trends; explain what they imply mechanistically.
2. Determine whether the training appears to include these phases, and justify each with evidence:
   - initial stable phase
   - effective unlearning phase
   - possible over-unlearning / catastrophic forgetting phase
3. Analyze the interaction among the main quantities, including:
   - total loss
   - loss_npo
   - anti loss
   - kl loss
   - neg_log_ratio
4. Assess whether the trade-off between bias reduction and language retention is reasonable, and identify the likely best checkpoint range.
5. Separate conclusions directly supported by the figures from expert inference.
6. Diagnose possible issues such as:
   - over-unlearning
   - reverse bias risk
   - weak KL regularization
   - overly strong anti loss
   - too many training steps
   - suboptimal LoRA target modules
   - learning-rate schedule not well suited for unlearning
7. Propose concrete next-round experiments, including:
   - which hyperparameters to change
   - suggested value ranges
   - which control or ablation experiments to add
   - which extra metrics should be monitored
8. If the evidence is insufficient, explicitly say what is missing instead of pretending certainty.

Constraints
- Do not invent numbers that are not visible in the figures or supplied logs.
- Do not treat correlation as causation without qualification.
- When a conclusion is inferential rather than directly supported, label it clearly as an inference.
- Prioritize four dimensions: bias reduction, LM retention, ICAT trade-off, and training stability.
- Rank recommendations by priority: what should be done first vs later.

Output format
Return the answer in this structure:
1. Executive summary
2. Plot-by-plot analysis
3. Training phase breakdown with evidence
4. Key issue diagnosis
5. Best checkpoint assessment
6. Next-round experiment recommendations (prioritized)
7. Suggested control / ablation studies
8. Uncertainty and missing information

Quality bar
Write as if this were the “results analysis + next experiment plan” section of a research paper. Be concrete, testable, and operational.

- The figures included are: [list plot names]
- Current configuration summary: [paste config]
- The main question I want answered is: [e.g., whether this is over-unlearning, where the best checkpoint is, which parameter to tune first]
